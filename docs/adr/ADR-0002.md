# ADR-0002: Engine Facade Isolation

## Status
Accepted

## Context
Exposing deep internal implementation details of packages, environments, and installers to external callers makes the system brittle. Changes to installer sub-modules could easily break the command-line interface (CLI) or script integrations.

## Decision
We will establish a unified `Engine` struct in `crates/anvil-core/src/api/v1.rs` as a stable programmatic API facade. The CLI and third-party consumers must route all queries and operations exclusively through this Engine.

## Consequences
- Internal structures can be refactored independently.
- The public API interface is stable, minimal, and compile-checked.
- Provides a clean separation of concerns between business logic and UI/CLI controllers.
