# ADR-0004: CLI Introspection Interface

## Status
Accepted

## Context
Diagnosing local environments requires introspection into past run executions, nested duration steps, active configurations, and live event streams. There was no standard CLI command set for querying local telemetry.

## Decision
We implement a dedicated CLI subcommand set `history`, `explain`, `trace`, and `events` that exposes local diagnostic information. These subcommands map directly to query functions on the public `Engine` facade.

## Consequences
- Developers get rich insights into the exact execution times of installers and sub-tasks.
- The interface is standardized and easy to invoke from developers' shells.
- Isolates diagnostic presentation logic strictly to the CLI package.
