# ADR-0003: In-process EventBus Hook

## Status
Accepted

## Context
When operations run, multiple internal stages publish status updates. Direct hook functions or hard-coded log statements coupling operations directly to printing formatting are inflexible.

## Decision
We implement a lightweight in-process `EventBus` using `tokio::sync::broadcast` that decouples event publishers from subscribers. The subscriber logging task acts as an in-process hook that listens to the stream and commits it asynchronously to disk.

## Consequences
- Operations remain unaware of how progress events are formatted, presented, or stored.
- Multiple consumers (such as CLI progress printers and journal logs) can listen to the event bus concurrently.
- Clean design aligned with event-driven architecture guidelines.
