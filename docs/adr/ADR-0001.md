# ADR-0001: Asynchronous Journal Storage

## Status
Accepted

## Context
Executing operations (e.g., sync, repair, clean) produces telemetry events. To inspect history, analyze execution errors, and trace nested operations, we need a local persistence mechanism. Writing directly from operation threads could block core execution on disk I/O, causing latency spikes.

## Decision
We will run journal logging in a non-blocking background tokio task spawned during `EventBus` initialization. Telemetry events broadcasted via the `EventBus` channel will be received by this subscriber task and appended asynchronously to the journal file.

## Consequences
- Main threads performing core engine work are shielded from disk writes.
- File system access is isolated inside the background logging task.
- Operates efficiently with low overhead in all runtime profiles.
