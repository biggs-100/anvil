# ADR-0006: Cache Integrity & Verification

## Status
Accepted

## Context
Checking the health of local runtimes requires scanning directories, checking for correct version lock files, verifying active configuration states, and testing for shim integrity. These steps should be fast and portable without locking the entire OS filesystem.

## Decision
We implement a lightweight polling file tailing system for live updates, coupled with dynamic filesystem integrity scanning during the `explain` and `inspect` queries. Rather than platform-native filesystem event subscriptions, portable file-seeking checks run loop-based polls.

## Consequences
- Highly portable across OS platforms including Windows, Linux, and macOS without extra native wrapper compilation.
- Low memory and execution foot-print.
- Avoids blocking issues or open file handle leaks during execution.
