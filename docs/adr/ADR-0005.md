# ADR-0005: Local JSON Lines Format

## Status
Accepted

## Context
A database engine or custom binary format introduces heavy external dependencies, database migrations, and complex C-bindings. The journal must be fast, append-only, human-readable, and crash-resilient.

## Decision
We select line-delimited JSON (NDJSON/JSON Lines) stored under `.forge/journal.jsonl` as the journal format. Each log record is written on its own line with a trailing newline.

## Consequences
- Highly portable across OSes and architectures.
- Easy to audit manually or stream using tools like `grep`, `jq`, and tail.
- Crash recovery is robust as partially written lines do not corrupt previous entries.
